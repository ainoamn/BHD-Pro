## 1) ملخص تنفيذي

المنتج **منشور ويعمل** على Vercel + Render + Neon، مع صفحة رئيسية ولوحة `/admin` ومسار محاسبة كامل للشركات.  
ما زالت فجوات واضحة في: **نضج لوحة تشغيل المنصة**، **استقرار الدومين عند بعض الشبكات**، و**صقل الهوية البصرية**.

| العنصر | الحالة الآن |
|--------|-------------|
| https://bhd-pro.vercel.app | يعمل |
| https://www.hisaby.pro | يعمل من بعض الشبكات (HTTP 200) ويتقطع عند أخرى بسبب مسار Vercel Anycast |
| https://hisaby.pro (بدون www) | سجل A قديم `76.76.21.21` — يحتاج تحديث إلى `213.188.79.1` |
| `/admin` | موجودة + محسنة؛ تتطلب تسجيل دخول مشرف |
| API `/api/admin/me` | موجودة (401 بدون توكن = المسار حي) |

## 2) أخطاء ومشاكل موثّقة

### أ) الدومين
- **ERR_TIMED_OUT / ERR_CONNECTION_ABORTED** على الهواتف أحيانًا رغم أن النشر صحيح.
- السبب: مسار شبكة إلى عناوين Vercel الجديدة (`216.198.79.x` / `64.29.17.x`) + سجل A للجذر ما زال على IP قديم.
- **الإجراء:** في Hostinger غيّر A لـ `@` إلى `213.188.79.1`، أو انقل DNS إلى Cloudflare Proxy.

### ب) Deploy Render
- فشل سابق `7054b9d` بسبب أخطاء TypeScript من `groupBy` على `SiteVisit`.
- تم الإصلاح بـ SQL خام + `prisma generate` في Docker.

### ج) لوحة `/admin`
- ظهور رسالة فشل تحقق عندما كان API قديمًا (404).
- تحويل لـ `/login` دون العودة لـ `/admin` — تم إصلاح `?next=`.
- ضعف التصميم والقدرات (حدود المستخدمين، تفاصيل الدفعات، تفاصيل المستخدم) — جاري تعزيزها في هذا الإصدار.

### د) Cold start
- Render Free يسبب بطء 10–50 ثانية في أول طلب بعد النوم.

### هـ) هوية بصرية
- المستخدم غير راضٍ عن الشعار/الصفحة الرئيسية/لوحة التحكم السابقة — تم استبدال الشعار وإعادة تصميم الصفحات (قابل لمزيد من التكرار حسب الذوق).

## 3) ما تم بناؤه وما يزال ناقصًا

### منجز
- محاسبة الشركة (فواتير، قيود، مخزون، تقارير، عملات…).
- نشر ثلاثي Vercel/Render/Neon.
- Google Sign-In.
- صفحة رئيسية ثنائية اللغة + دخول/تسجيل.
- لوحة منصة: مؤشرات، شركات، مستخدمون، مدفوعات، عروض، زيارات، بوابات.
- حدود مستخدمين/فواتير قابلة للتجاوز لكل شركة (`usersLimitOverride` / `invoicesLimitOverride`).
- تفاصيل مستخدم: شركة، اشتراك، مدفوعات، جلسات IP.

### ناقص / يحتاج تطوير

#### أولوية عالية
1. **استقرار الدومين** (DNS/Cloudflare) واختبار من شبكات عمان المختلفة.
2. **لوحة تشغيل أغنى:** فلاتر متقدمة، تصدير CSV، رسوم بيانية، إشعارات.
3. **تطبيق أكواد الخصم فعليًا** في مسار الدفع (العروض تُحفظ لكن الربط الكامل مع checkout يحتاج إكمال).
4. **ترقية Render** أو keep-warm لتقليل البطء.
5. **تدوير الأسرار** التي ظهرت أثناء الإعداد.

#### أولوية متوسطة
6. GeoIP حقيقي للزيارات (بدل افتراض OM).
7. صلاحيات دقيقة داخل لوحة المنصة (أكثر من مشغّل بأدوار).
8. صفحات تسويقية إضافية (أسعار مستقلة، مدونة، مركز مساعدة) بمستوى Wafeq.
9. استبدال دردشة AI الوهمية بتكامل حقيقي أو إخفاؤها.
10. فرض حدود الاشتراك في كل المسارات الحساسة بشكل موحّد + اختبارات.

#### أولوية لاحقة
11. تطبيقات جوال / API عام موثّق للشركاء.
12. تقارير منصة: MRR/ARR/churn.
13. نسخ احتياطي مجدول ومراقبة (Sentry/Uptime).
14. تعدد العلامات التجارية white-label.
